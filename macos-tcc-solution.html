CLI 工具 macOS TCC 权限方案

基于 open-codex-computer-use 项目的实践总结。解决 CLI 工具在 macOS 上 Accessibility / Screen Recording 权限授权不稳定、版本更新后丢失的问题。

问题根因

  1. 没有 bundle ID — TCC 按文件路径匹配，路径不稳定（符号链接、不同包管理器路径不同），经常匹配失败
  2. ad-hoc 签名每次变化 — 版本更新后 code identity 变了，TCC 不认识新版本，每次都要重新授权
  3. 用户手动操作不可靠 — 去系统设置里找隐私列表、点加号、手动拖入，流程太长，用户经常搞不定

方案: .app bundle 结构 + Apple Development 免费签名 + 拖拽引导 UI + Unix socket agent proxy。

架构

用户: your-tool <command>
        |
        v
  your-tool-cli (CLI 壳，无权限)
        |  Unix domain socket
        v
  YourTool.app agent 进程 (持 Accessibility + Screen Recording)
        |
        v
  返回结果 -> stdout

5. install.sh

首次安装脚本：复制 .app 到 /Applications，注册 Gatekeeper 信任，创建 CLI 软链接。

#!/usr/bin/env bash
set -euo pipefail
APP_NAME="YourTool"
INSTALL_DIR="/Applications/${APP_NAME}.app"

cp -R "$(dirname "$0")/${APP_NAME}.app" "${INSTALL_DIR}"
sudo spctl --add --label "${APP_NAME}" "${INSTALL_DIR}"
sudo ln -sf "${INSTALL_DIR}/Contents/MacOS/your-tool-cli" "/usr/local/bin/your-tool"
echo "Done. Run: your-tool doctor"

spctl --add --label 是核心 — 告诉 Gatekeeper 信任此 app，Apple Development 证书签的 .app 首次启动不弹警告。只在首次安装执行一次。

6. OSS 分发

CI 产出 your-tool-{version}-macos.zip，含 YourTool.app/ 和 install.sh。

关于 Apple Development 免费证书

  • 费用: 免费 Apple ID 即可，不需要 $99/年的 Apple Developer Program
  • 获取: developer.apple.com/account -> Certificates -> Apple Development
  • TCC 稳定性: 同一张证书签名所有版本，TCC 权限永久保持
  • 局限: 证书有效期 1 年，到期需续签重发
  • 与 ad-hoc 对比: ad-hoc 每次 rebuild 身份变化 -> 每次更新都要重授权；Apple Development 稳定一年，体验质变
